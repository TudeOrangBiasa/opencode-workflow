Workflow Flow (Planner → Execution → Verify → Review)

User Request → Planner: Plan approach → Load skills + prior lessons → Advisor: validate plan (scope, drift) → Delegate to builder → Builder: implement + verify → Planner: verify-evidence gate → Delegate to reviewer → Reviewer: code + browser QA → Advisor: judge output (drift, quality) → Report / retry / done → Eval: advisor reviews session

🔌 3. Plugin System

Two npm plugins hook into OpenCode runtime. No repo-local plugins.

🦣 opencode-caveman

npm v0.1.4
PurposeToken-compression mode: strips articles, filler, hedging. Forces terse communication
Levelslite · full (default) · ultra · wenyan
Skill1 skill: caveman/SKILL.md

🐴 opencode-ponytail

npm v4.7.3
PurposeLazy senior dev: stdlib first, delete over-engineering, shortest diff wins
Hooksconfig · system.transform · command.execute.before
Levelslite · full (default) · ultra
Skills5 skills: main, audit, debt, help, review
Commands/ponytail {level} · /ponytail-audit · /ponytail-review

Plugin Architecture

📥 Config Hook

Registers slash commands at startup. Parses markdown from command/ directory.

🔄 System Transform

Injects caveman/ponytail ruleset into system prompt every turn. Reads state from ~/.config/opencode/.ponytail-active.

⚡ Command Execute

Intercepts /ponytail commands, persists intensity level, invokes skill.

📦 6. Deployment Flow

opencode-workflow → scripts/link-skills.sh → ~/.config/opencode/skills/ → opencode.json → OpenCode Runtime → Restart → ✅ Active
StepWhat Happens
1. DevelopEdit agent files in agents/, skills in skills/
2. Linkscripts/link-skills.sh creates categorized bucket symlinks in ~/.config/opencode/skills/
3. Config~/.config/opencode/opencode.json points paths, model routing, plugins, MCP
4. ActivateRestart OpenCode → loads agents, skills, plugins, MCP
5. RunPlanner receives request → advisor validates plan → delegates to builder → reviewer QA → advisor judge → eval

🎯 7. Goals & Integration Destinations

📈 Session Eval

Destination: Eval reports in .scratch/evals/ + OpenViking lessons
Via: Advisor runs post-session eval → memory-dreaming consolidates
Tools: eval skill, openviking MCP, memory-dreaming

🎨 Design Output

Destination: Frontend code, design tokens, CSS, component code
Via: planner → designer skill → generates UI code + visual specs
Tools: design-skill (22 sub-commands), chrome-devtools for validation

📄 Document Output

Destination: .docx reports, .pptx decks, academic papers
Via: planner → documents-kit → officecli creates/edits files
Tools: officecli MCP, drawio diagrams, humanizer for prose

🔧 Code Output

Destination: Production code, tests, migrations
Via: planner → builder → reviewer cycle
Tools: tdd, diagnosing-bugs, security-review, php-review

🧠 Learning & Memory

Destination: OpenViking persistent store
Via: Every agent stores lessons via ov add-memory
Tools: openviking MCP, memory-dreaming skill for consolidation

📊 Research Output

Destination: Research docs, reports, synthesis
Via: scout (exa web search) + explorer (codebase) → advisor synthesizes
Tools: exa MCP, deep-research skill, handoff skill

📋 Issue Management

Destination: Local markdown issues in .scratch/issues/
Via: to-spec → to-tickets → triage pipeline
Tools: triage skill, issue-tracker docs, wayfinder for exploration
