# Domain Docs

Use `CONTEXT.md` for shared language and `docs/adr/` for decisions.

## CONTEXT.md

- Define terms agents should use exactly.
- Mark ambiguous or retired terms.
- Keep definitions short and operational.

## ADRs

- Store ADRs in `docs/adr/`.
- Use numbered filenames like `0001-short-title.md`.
- Record context, decision, consequences, and status.

## Workflow

- Use `grill-with-docs` when terminology or decisions are unclear.
- Update docs only when a decision has stabilized.
