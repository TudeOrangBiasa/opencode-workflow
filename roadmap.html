✂️ Issue 24-25 — Hashline Edit Phase 3 ~750 lines TS

How Hashline Works

[greet.py#A1B2]
SWAP 2.=2:
+ greeting = "Hi"
+ msg = f"{greeting}, {name}"
⇧ 85 chars vs unified diff 130 chars (~35% savings).
61% avg savings with block ops.
9 Operations
OpSyntax
SWAPSWAP N.=M: + body
SWAP.BLKSWAP.BLK N: + body
DELDEL N.=M
DEL.BLKDEL.BLK N
INS.PREINS.PRE N: + body
INS.POSTINS.POST N: + body
INS.HEADINS.HEAD: + body
INS.TAILINS.TAIL: + body
REM/MVREM / MV dest

Execution Pipeline

StageFileFunction
1. Parseparser.tsSplit input on [path#TAG] headers → PatchSection[]
2. Tokenizetokenizer.tsLine-by-line: headers, ops, body rows
3. Executeexecutor.tsState machine: ops → Edit[] with boundary repair
4. Preparepatcher.tsRead file, verify tag, apply edits in memory
5. Commitpatcher.tsWrite via Filesystem + LSP writethrough

Key Features

🔐 Content Hash

16-bit xxHash32 of file. #A1B2 tag validates file before edit. Collision handled by full-text equality.

👁️ Seen-Line Guard

Rejects edits on lines model never read. Prevents off-by-memory corruption.

🔧 Boundary Repair

Delimiter-balance analysis fixes model off-by-one mistakes (braces/brackets/parens).

♻️ 3-Way Recovery

If file drifted since read: apply to cached version, diff, merge onto live content.
